## 系统架构

### 后端架构
- **API风格**: RESTful API
- **认证方式**: JWT Token
- **权限控制**: 基于角色的访问控制（RBAC）
- **工作流**: 自定义工作流引擎

### 前端架构
- **组件化**: 函数式组件 + Hooks
- **类型安全**: 完整的 TypeScript 类型定义
- **状态管理**: 全局状态（Context）+ 服务器状态（React Query）+ 本地状态（Hooks）
- **路由保护**: 基于角色的路由守卫

## 开发规范

### 后端规范
- 使用类型提示（Type Hints）
- 遵循 PEP 8 代码规范
- API 使用 RESTful 风格
- 所有数据库操作使用 ORM，避免直接 SQL

### 前端规范
- 使用 TypeScript 进行类型检查
- 遵循 ESLint 代码规范
- 组件化开发，提高代码复用性
- 使用 React Query 进行数据获取和缓存管理

## 部署要求

### 后端
- Python 3.8+
- MySQL 5.7+ 或 8.0+
- 建议使用虚拟环境（venv）

### 前端
- Node.js 16+
- npm 或 yarn 或 pnpm
- 生产环境需要构建（npm run build）
