## 6. API架构设计

### 6.1 RESTful API 端点

```
API Base URL: /api

认证相关
├─ POST   /auth/register          用户注册
├─ POST   /auth/login             用户登录
├─ POST   /auth/logout            用户登出
└─ GET    /auth/me                获取当前用户信息

用户管理
├─ GET    /users                  用户列表
├─ GET    /users/:id              用户详情
├─ POST   /users                  创建用户

├─ DELETE /users/:id              删除用户
└─ POST   /users/:id/approve      审批用户

### 6.2 认证与授权流程

```
┌──────────┐                                    ┌──────────┐
│  Client  │                                    │  Server  │
└─────┬────┘                                    └────┬─────┘
      │                                              │
      │  1. POST /api/auth/login                     │

      ├─────────────────────────────────────────────>│
      │                                              │
      │                                   2. 验证用户凭证
      │                                              │
      │                            3. 生成JWT Token  │
      │  4. { token, user_info }                     │
      │<─────────────────────────────────────────────┤
      │                                              │
      │  5. 保存Token到本地存储                      │
      │                                              │
      │  6. GET /api/resource                        │
      │  Authorization: Bearer <token>               │
      ├─────────────────────────────────────────────>│
      │                                              │
      │                            7. 验证Token有效性  │
      │                                              │
      │                         8. 检查用户权限(RBAC) │
      │                                              │
      │  9. { data }                                 │
      │<─────────────────────────────────────────────┤
      │                                              │

     │ 提交任务
     ↓

└────┬─────┘            │
     │                  │ 修改后回退
     ├─────→ 总管审批    │
     │                  │
     ├─ 确认 ──→ ┌──────────┐
     │           │CONFIRMED │  已确认
     │           │ (已审批)  │
     │           └────┬─────┘
     │                │
     │                │ 销售接口人提交详细需求
     │                ↓
     │           ┌──────────────┐
     │           │DETAIL_SUBMITTED│ 详细需求已提交
     │           └────┬───────────┘
     │                │
     │                │ 总管派单
     │                ↓
     │           ┌──────────┐
     │           │DISPATCHED│  已派单
     │           │ (全部派单)│
     │           └────┬─────┘
     │                │
     │                │ 成员接单
     │                ↓
     │           ┌──────────┐
     │           │IN_PROGRESS│ 进行中
     │           │ (执行中)   │
     │           └────┬─────┘
     │                │
     │                │ 所有工单完成
     │                ↓
     │           ┌──────────┐
     │           │COMPLETED │  已完成
     │           └──────────┘
     │
     └─ 拒绝 ──→ ┌──────────┐
                 │ REJECTED │  已拒绝
                 └──────────┘

     任意状态 ──→ ┌──────────┐
     (总管操作)   │CANCELLED │  已关闭
